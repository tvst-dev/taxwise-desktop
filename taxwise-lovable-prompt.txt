Card Flow (all API calls go to your server-side routes with PAYSTACK_SECRET_KEY):
  1. POST /api/paystack/charge-card with { email, amount_in_kobo, card: {number, cvv, expiry_month, expiry_year}, metadata }
  2. Handle Paystack response status:
       "send_pin"  → show PIN entry screen (large centered 4-digit PIN input, password type)
       "send_otp"  → show OTP entry screen (6-digit input, show display_text as hint)
       "open_url"  → show "Open Authentication Page" button (opens URL in new tab) + poll every 5s + "I've completed authentication" submit button
       "success"   → call finalize(reference)
       anything else → show gateway_response as red error toast, stay on card screen
  3. PIN screen: POST /api/paystack/submit-pin with { reference, pin } → handleChargeResponse again
  4. OTP screen: POST /api/paystack/submit-otp with { reference, otp } → handleChargeResponse again

Bank Transfer Flow:
  1. POST /api/paystack/init-transaction with { email, amount_in_kobo, channels: ['bank_transfer'], metadata }
     Response: { authUrl, reference }
  2. Open authUrl in a modal popup/iframe window (Paystack shows the virtual account number to transfer to)
  3. Simultaneously start polling GET /api/paystack/verify/:reference every 5 seconds in the background
  4. When poll detects status === "success":
       - Close the popup window programmatically
       - Call finalize(reference)
  5. If user closes popup before payment confirmed:
       - Do one final verify check
       - If confirmed: finalize(reference)
       - If not confirmed: show toast "Transfer cancelled or not yet confirmed", reset to method screen

finalize(reference) function:
  1. GET /api/paystack/verify/:reference — confirm status === "success"
  2. UPDATE organizations SET subscription_status='trial', subscription_tier=selectedPlan,
     trial_started_at=now(), trial_ends_at=now()+14days WHERE id=organizationId
  3. UPSERT subscriptions (organization_id, plan, status='active', paystack_reference, period dates)
  4. Immediately update in-memory auth store: updateOrganization({ subscription_status: 'trial', subscription_tier: plan, trial_ends_at })
  5. Immediately apply feature flags for the new tier (do NOT require re-login)
  6. Show success view

TAX CALCULATION LOGIC (Nigeria — implement exactly)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─── PAYE (Pay As You Earn) ───────────────────────────────────────────────────

Step 1: Compute Gross Income
  gross = basic salary + housing allowance + transport allowance + other allowances

Step 2: Compute Statutory Deductions (reduce taxable income)
  Pension    = 8% of (basic + housing + transport), minimum NGN 1 — employee contribution
  NHF        = 2.5% of basic salary
  NHIS       = 5% of gross (optional, user toggles)
  Life Insurance = actual premium paid (optional, user enters amount)

Step 3: Compute Gross Taxable Income (GTI)
  GTI = gross income - total statutory deductions

Step 4: Compute Consolidated Relief Allowance (CRA)
  CRA = higher of (NGN 200,000 OR 1% of GTI)  +  20% of GTI

Step 5: Compute Taxable Income
  taxable_income = GTI - CRA
  (if taxable_income < 0, set to 0)

Step 6: Apply Progressive Tax Bands (annual amounts)
  First  NGN 300,000      → 7%
  Next   NGN 300,000      → 11%
  Next   NGN 500,000      → 15%
  Next   NGN 500,000      → 19%
  Next   NGN 1,600,000    → 21%
  Above  NGN 3,200,000    → 24%

Step 7: Monthly PAYE = annual tax / 12

Display full breakdown:
  Gross Income, Pension deduction, NHF deduction, NHIS deduction, Life Insurance deduction,
  Total Deductions, GTI, CRA calculation (show both options and which was higher),
  Taxable Income, tax per band (show each band and amount applied),
  Total Annual Tax, Monthly PAYE, Take-home Pay (gross - pension - NHF - NHIS - monthly PAYE),
  Effective Tax Rate %


─── CIT (Companies Income Tax) ──────────────────────────────────────────────

Only applicable to Limited Companies.

Company size by annual turnover:
  Small   (turnover < NGN 25,000,000/year):          0% CIT (exempt)
  Medium  (NGN 25,000,000 to NGN 100,000,000/year): 20% CIT
  Large   (turnover > NGN 100,000,000/year):         30% CIT

Education Tax (applies to all companies regardless of size): 2.5% of assessable profit

Steps:
  1. Inputs: total annual revenue, total allowable expenses
  2. Profit before tax = revenue - expenses
  3. Determine company size band from revenue
  4. CIT = profit before tax x applicable CIT rate
  5. Education Tax = profit before tax x 2.5%
  6. Total Tax Payable = CIT + Education Tax

Display: Revenue, Expenses, Profit Before Tax, Company Size Category,
  CIT Rate, CIT Amount, Education Tax (2.5%), Total Tax Payable, Effective Rate %


─── VAT (Value Added Tax) ───────────────────────────────────────────────────

Rate: 7.5% (Nigeria Finance Act 2020)

Mode A — Simple Sales VAT:
  User enters sales amount and chooses: VAT-exclusive or VAT-inclusive
  VAT-exclusive: VAT = amount x 0.075 | Total Invoice = amount + VAT
  VAT-inclusive: VAT = amount x (0.075 / 1.075) | Net = amount - VAT

Mode B — VAT Return (Input/Output):
  Output VAT = total VAT charged to customers on sales
  Input VAT  = total VAT paid on business purchases
  Net VAT Payable = Output VAT - Input VAT
  (positive = remit to FIRS; negative = VAT credit/refund claim)

Filing frequency: monthly
Display: Period (month/year), Output VAT, Input VAT, Net VAT Payable/Refundable,
  Filing deadline (21st of following month)


─── WHT (Withholding Tax) ────────────────────────────────────────────────────

WHT rates by payment/transaction type:
  Dividends                               10%
  Interest                                10%
  Rent (companies)                        10%
  Rent (individuals)                      10%
  Royalties                               10%
  Directors fees                          10%
  Commissions (companies)                 10%
  Commissions (individuals)               5%
  Management / Consulting fees            10%
  Technical / Professional services fees  10%
  Construction contracts                  5%
  Supply of goods / materials             5%
  Agency arrangements                     5%

Inputs:
  Transaction type (dropdown with all types above)
  Gross payment amount
  Payer type (Individual / Company)

Outputs:
  WHT Amount = gross x rate
  Net Payment After WHT = gross - WHT
  Remittance deadline = 21st of the month following deduction


─── Stamp Duty ──────────────────────────────────────────────────────────────

Tenancy / Lease Agreements:
  Up to 7 years:       0.78% of total rent
  7 to 21 years:       3% of total rent
  Above 21 years:      6% of total rent

Share Transfer:        1.5% of transaction value

Receipt / Acknowledgment (amount > NGN 10,000):  NGN 50 flat fee


─── Auto-Create Reminders after saving calculations ─────────────────────────

After user saves any tax calculation, auto-create a reminder with:
  VAT calculation saved  → reminder: "VAT Filing Due" on 21st of following month, priority=high
  WHT calculation saved  → reminder: "WHT Remittance Due" on 21st of following month, priority=high
  PAYE calculation saved → reminder: "PAYE Remittance Due" on 10th of following month, priority=high
  CIT calculation saved  → reminder: "CIT Filing Due" on last day of 6th month after company year-end, priority=critical

SERVER-SIDE API ROUTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

All Paystack API calls MUST go through server-side routes.
PAYSTACK_SECRET_KEY must NEVER be sent to or accessible from the browser.

POST /api/paystack/init-transaction
  Body: { email, amount (NGN), channels (array), metadata (object) }
  Server: converts amount to kobo (x100), calls Paystack /transaction/initialize
  Returns: { status, authUrl, accessCode, reference }

POST /api/paystack/charge-card
  Body: { email, amount (kobo), card: { number, cvv, expiry_month, expiry_year }, metadata }
  Server: calls Paystack /charge
  Returns Paystack response as-is

POST /api/paystack/submit-pin
  Body: { reference, pin }
  Server: calls Paystack /charge/submit_pin
  Returns Paystack response as-is

POST /api/paystack/submit-otp
  Body: { reference, otp }
  Server: calls Paystack /charge/submit_otp
  Returns Paystack response as-is

GET /api/paystack/verify/:reference
  Server: calls Paystack /transaction/verify/:reference
  Returns transaction data

SUPABASE EDGE FUNCTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Function: send-invite
  Triggered by frontend (anon key auth — use SUPABASE_ANON_KEY as Bearer token)
  Inputs: { email, role, organizationId, organizationName, inviterName }
  Process:
    Use SUPABASE_SERVICE_ROLE_KEY to call supabase.auth.admin.generateLink({ type: 'invite', email, options: { redirectTo: 'https://www.taxwise.com.ng', data: { organization_id, role, ... } } })
    If user already exists and unconfirmed: delete user, retry generateLink
    If user already exists and confirmed: return 409 error
    Send styled HTML email via Resend API
    Upsert record in team_invitations table
  Email: dark theme HTML, TaxWise logo, inviter name, org name, role, "Accept Invitation" button

Function: extract-document
  Triggered by frontend (user access_token auth)
  Inputs: { imageBase64, mimeType, documentType }
  Process:
    Calls OpenAI GPT-4o with the image and a structured extraction prompt
    Returns extracted fields as JSON based on document type
  Requires: OPENAI_API_KEY env var

Function: chat-assistant
  Triggered by frontend (user access_token auth)
  Inputs: { messages (last 10), context: { businessType, subscriptionTier, currentMonthIncome, currentMonthExpenses } }
  Process:
    Builds system prompt with Nigerian tax expertise + user's business context
    Calls OpenAI GPT-4o with messages array
    Returns { reply: string }
  Requires: OPENAI_API_KEY env var
