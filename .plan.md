# TaxWise Desktop — Comprehensive Project Understanding

## 1. Project Overview
TaxWise is a **Nigerian Tax Management & Compliance System** built as an **Electron desktop application** with a React frontend. It targets Nigerian businesses and individuals, handling PAYE, CIT, VAT, WHT, CGT, and Education Tax calculations. The app integrates with **Supabase** for backend services (auth, database, edge functions) and **Paystack** for subscription payments.

## 2. Technology Stack

| Layer | Technology |
|-------|------------|
| **Frontend** | React 18, React Router v6 (HashRouter), react-hot-toast |
| **State Management** | Zustand (with persist middleware for localStorage) |
| **Desktop Shell** | Electron 27, electron-builder, electron-updater |
| **Build** | Webpack 5, Babel, CSS inline styles (no Tailwind/styled-components) |
| **Backend** | Supabase (PostgreSQL + Auth + Edge Functions) |
| **Payments** | Paystack (via local proxy server + Supabase Edge Function) |
| **Local DB** | sql.js (SQLite WASM — pure JS, no native compilation) |
| **AI** | OpenAI GPT-4o (document extraction via main process or edge function) |

## 3. Architecture

### 3.1 Entry Points
- **`main.js`** — Electron main process. Creates BrowserWindow, manages deep links (`taxwise://`), handles IPC, runs local SQLite via sql.js, starts Paystack proxy server.
- **`preload.js`** — Secure context bridge exposing only whitelisted APIs to renderer.
- **`src/index.jsx`** → renders `App.jsx` inside React.StrictMode.

### 3.2 Routing (`App.jsx`)
Uses `HashRouter` (ideal for Electron `file://` protocol):
- **Public routes**: `/login`, `/register`, `/forgot-password`, `/reset-password`
- **Protected routes** (under `Layout`): `/dashboard`, `/entries`, `/analytics`, `/cash-flow`, `/calculator`, `/history`, `/deductions`, `/reminders`, `/audit`, `/documents`, `/pos`, `/api`, `/team`, `/settings`

### 3.3 Auth & Subscription Gate
- `ProtectedRoute`: checks `isAuthenticated` + org subscription status must be `trial` or `active`.
- `PublicRoute`: redirects authenticated users with valid subscriptions to `/dashboard`.
- `SubscriptionExpired` component shown for non-active/trial users.

### 3.4 State Management (Zustand with localStorage persistence)
| Store | Purpose |
|-------|---------|
| `useAuthStore` | User, org, auth state, RBAC permissions/roles |
| `useEntriesStore` | Financial entries with filtering & pagination |
| `useTaxStore` | Tax calculations (preserves local-only fallback records) |
| `useDeductionsStore` | Deductions with merge logic for local fallbacks |
| `useRemindersStore` | Reminders with upcoming/overdue helpers |
| `usePOSStore` | Products, cart, transactions; normalizes DB columns (`stock_quantity` → `stock`) |
| `useBankStore` | Bank accounts, transactions |
| `useTeamStore` | Members, invitations |
| `useDocumentsStore` | Document metadata (file objects stripped from persistence) |
| `useFeaturesStore` | Feature flags gated by subscription tier |
| `useUIStore` | Sidebar, modals (with stack), notifications, global loading |
| `useDashboardStore` | Stats, date ranges |
| `useSettingsStore` | General, notifications, integrations, display settings |

## 4. Backend Integration

### 4.1 Supabase (`src/services/supabase.js`)
- **Auth**: signUp, signIn, signOut, onAuthStateChange, deep link handling (`taxwise://` tokens → `setSession`).
- **CRUD operations** for: organizations, users, entries, tax_calculations, deductions, documents, bank accounts, POS products/transactions, reminders, team invitations.
- **Edge function callers**: `extractDocument`, `initializePayment`, `verifyPayment`, `chatWithAssistant`.
- **`sendTeamInvite`** intentionally uses the **anon key as Bearer token** (not the user's access token) to avoid "Invalid JWT" errors when the user session is stale.

### 4.2 Data Sync (`src/services/dataSync.js`)
- **`loadOrganizationData(orgId)`** — loads all org data from Supabase into Zustand stores.
- **Graceful degradation** — each store loads independently; one failure never blocks others.
- **Local fallback preservation** — tax calculations and deductions with locally-generated IDs (`calc_*`, `ded_*`) are preserved and merged with DB data.
- **`clearLocalData()`** — wipes all stores on logout.
- **Individual sync helpers** for create/update/delete that update both Supabase and local store atomically.

### 4.3 Edge Functions
| Function | Purpose |
|----------|---------|
| `paystack/index.ts` | initialize, charge_card, submit_otp, verify, create_subscription, charge_authorization, get_subscription, cancel_subscription, webhook handling |
| `extract-document/index.ts` | AI document extraction using GPT-4o; converts foreign currencies to NGN; supports PDF (via `type: "file"`) and images (via `type: "image_url"`) |
| `chat-assistant/index.ts` | AI chat assistant |
| `send-invite/index.ts` | Team invitation emails |
| `accept-invite/index.ts` | Invite acceptance |
| `mono/index.ts` | Bank integration (Mono) |
| `verify-api-key/index.ts` | API key verification |

## 5. Subscription & Payment System

### 5.1 Plans
| Plan | Price | Key Limits |
|------|-------|-----------|
| **Startup** | ₦10,000/mo | 1 user, 50 calculations, 10 doc uploads, no POS/bank |
| **SME** | ₦25,000/mo | 5 users, 100 calculations, 50 doc uploads, POS + 1 bank account |
| **Corporate** | ₦60,000/mo | Unlimited users, calculations, docs, API access, multi-bank |

### 5.2 Payment Flow (Registration)
1. **4-step wizard**: Account → Business → Plan → Card
2. **Card verification**: ₦100 charged and immediately refunded (Paystack minimum)
3. **14-day trial** activated after successful card verification
4. After trial: subscription auto-charges via saved `authorization_code`
5. Subscription statuses: `trial`, `active`, `pending`, `past_due`, `expired`, `cancelled`

### 5.3 Payment Infrastructure
- **`server.js`** — Local Express proxy on port 5555 inside Electron main process. Holds `PAYSTACK_SECRET_KEY` from `.env`. Never exposes secret to renderer.
  - Endpoints: `/api/init-transaction`, `/api/verify/:reference`, `/api/charge-card`, `/api/submit-pin`, `/api/submit-otp`, `/api/init-bank-transfer`, `/api/subscribe`
- **`src/components/Payment/PaymentForm.jsx`** — Full inline payment UI (no Paystack popup).
  - Steps: `method` → `card` → `pin` → `otp` → `verifying` → `success`
  - Also supports **bank transfer** via Paystack popup (`window.electronAPI.payment.openPopup`) + background polling every 5s for 6 minutes.
  - After successful payment, updates Supabase `organizations` table and calls `applyFeatureFlags(plan)` to immediately unlock features.
- **`supabase/functions/paystack/index.ts`** — Backend Paystack edge function with webhook handling, subscription management, and auto-refund for trial verification charges.

## 6. Tax Calculator (`src/services/taxCalculator.js`)

### 6.1 Supported Tax Types
| Tax | Description |
|-----|-------------|
| **PAYE** | Personal Income Tax for individuals. Uses 6 progressive bands (7%–24%). CRA = MAX(₦200k, 1% gross) + 20% gross. |
| **CIT** | Company Income Tax. Rates: 0% (≤₦25m), 20% (₦25m–₦100m), 30% (>₦100m). Plus 2.5% Education Tax on assessable profit. |
| **VAT** | Value Added Tax at 7.5%. Output VAT − Input VAT − Credit B/F. |
| **WHT** | Withholding Tax. Rates vary by transaction type (5%–10%) and recipient type (individual/company). |
| **CGT** | Capital Gains Tax at 10% on chargeable gain. |
| **Monthly PAYE** | Calculates monthly payroll PAYE from salary components. |

### 6.2 Key Features
- Auto-populate from entries for CIT, VAT, and PAYE.
- Fiscal year filtering (parses `YYYY-MM-DD` strings as local time to avoid UTC off-by-one-day issues).
- Results saved to `tax_calculations` table with fallback to local storage if DB fails.

## 7. Document AI Extraction (`src/components/Documents/Documents.jsx` + Edge Function)
- Upload PDF, JPG, PNG, WebP, XLSX, CSV (max 50MB).
- AI extracts: vendor, amounts, dates, line items, currency detection.
- **Foreign currency conversion** to NGN using hardcoded exchange rates (USD→1550, EUR→1680, GBP→1950, etc.).
- Extracted data can be converted directly into an **Entry** (income/expense) + auto-creates a **Deduction** for expense entries.
- Blob URLs stored in-memory per session for viewing uploaded files.

## 8. Electron Main Process (`main.js`)

### 8.1 Key Features
- **Single instance lock** with deep link forwarding.
- **Custom protocol**: `taxwise://` for auth callbacks (invite acceptance, password recovery).
- **Paystack payment popup**: `ipcMain.handle('payment:openPopup', ...)` opens a child BrowserWindow, intercepts redirect to `localhost:52731` to extract reference.
- **Auto-updater**: GitHub releases, checks silently on launch, notifies renderer of available updates.
- **SQLite via sql.js**: Pure JS SQLite, no native modules. Database saved to `userData/taxwise.db` in production, auto-saves every 30s.
- **Secure OpenAI handlers**: API key stored encrypted in `electron-store`, document extraction happens in main process (key never reaches renderer).
- **Window controls**: minimize, maximize, close via custom title bar.

### 8.2 IPC Channels Exposed via Preload
- `window` (minimize/maximize/close)
- `db` (query/run/get/all for SQLite)
- `file` (save/open/saveBuffer)
- `shell` (openExternal/openPath)
- `app` (getVersion/getPath)
- `onDeepLink` (subscribe to deep links)
- `store` (encrypted key-value storage)
- `ai` (hasApiKey/setApiKey/removeApiKey/extractDocument/getUsage)
- `payment` (openPopup/closePopup)
- `updates` (check/install/onUpdateAvailable/onUpdateDownloaded)

## 10. Database Schema Highlights (SQLite/PostgreSQL)
- **organizations** — subscription state, Paystack tokens, trial dates, card info (last4/brand/expiry)
- **users** — profiles with roles: owner, admin, manager, accountant, cashier, viewer
- **entries** — income/expense/transfer/liability/asset with VAT, WHT, approval workflow
- **tax_calculations** — computed results with JSON input_data and breakdown
- **deductions** — pension, NHF, NHIS, life insurance, capital allowances
- **documents** — upload metadata, AI extraction status, extracted_data JSON
- **pos_products, pos_sales, pos_sale_items** — full POS system
- **bank_accounts, bank_transactions** — Mono bank integration support
- **team_invitations** — pending invites with tokens and expiry
- **audit_logs** — full activity tracking with old/new values
- **ai_insights** — AI-generated tax tips and recommendations

## 11. Security & Permissions
- **RBAC**: 6 roles with granular permissions (owner/admin/manager/accountant/cashier/viewer)
- **Feature flags** gated by subscription tier via `applyFeatureFlags(tier)`
- **Supabase RLS** policies configured in migrations
- **Electron contextIsolation** enabled, `nodeIntegration` disabled
- **Paystack secret** only in main process / edge function, never in renderer
- **OpenAI API key** encrypted in `electron-store`, accessed only via main process IPC

## 12. Notable Implementation Details
- **Deep link support**: `taxwise://` protocol for invites and password recovery. Main process parses tokens and sends to renderer.
- **Session keepalive**: pings Supabase session every 10 minutes to prevent 1-hour expiry.
- **Offline-first**: Zustand stores persist to localStorage; syncs to Supabase when online.
- **Graceful degradation**: Each data loader in `dataSync.js` is wrapped in individual try/catch.
- **Auto-updater**: Uses GitHub releases with optional `GH_TOKEN` for private repos.
- **Document extraction fallback**: If Supabase edge function fails, main process can extract via direct OpenAI call.
- **Currency conversion**: Foreign invoices automatically converted to NGN using hardcoded rates in `extract-document` edge function.

## 13. Potential Areas for Fixes (Observed)
1. **`src/services/paystack.js`** imports `React` unnecessarily in a plain JS service file.
2. **Config (`src/config.js`)** has hardcoded API keys and secrets (should be environment-based).
3. **`preload.js`** exposes `supabase` IPC channels (signIn/signUp/signOut/getUser/etc.) that are **not implemented** in `main.js` — this could cause runtime errors if called.
4. **Exchange rates** in `extract-document/index.ts` are hardcoded and will become stale.
5. **`featureFlags.js`** references `enterprise` tier but app uses `corporate` — mapping mismatch.
6. **Paystack edge function** `handleChargeCard` calls `/charge` directly without initializing a transaction first, which may not work for all card types without prior `/transaction/initialize`.
7. **`TaxCalculator.jsx`** auto-calculate `useEffect` only triggers for CIT, not other tax types.
8. **`RegisterPage.jsx`** uses `window.electronAPI.payment.openPopup` in bank transfer path but the `PaymentForm.jsx` handles its own bank transfer via local server API — potential inconsistency.
